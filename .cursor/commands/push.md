# 提交代码到远程

## 概述
将当前工作区所有改动提交并推送到远程 `origin main` 分支。

## 步骤
1. **查看状态**：执行 `git status`，确认有哪些改动。
2. **确定提交说明**：
   - 若用户在输入本命令时在命令名后写了内容（例如 `/push feat: 添加 AI 侧边栏`），则将该内容作为提交说明（commit message）。
   - 若未提供，则使用默认提交说明：`chore: update`。
3. **暂存**：执行 `git add -A`。
4. **提交**：执行 `git commit -m "<提交说明>"`，提交说明需用双引号包裹，若包含双引号则转义。
5. **推送**：执行 `git push origin main`。

## 注意
- 若 `git status` 无改动，提示用户「当前没有可提交的改动」并结束，不要执行 commit。
- 若 `git commit` 失败（例如无改动或未暂存），根据终端输出给出简短说明，不要继续执行 push。
- 提交说明尽量符合常见规范（如 feat/fix/docs/chore: 描述）。
